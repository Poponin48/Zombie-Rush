# **Зомби**

## **Статус реализации (зафиксировано)**

### **Сделано**

* Префаб / настройка зомби под игровую логику (компоненты на объекте).  
* **NavMesh** — запечённая навигация по уровню, зомби перемещаются по сетке.  
* **Преследование** — после входа игрока в радиус активации зомби бежит к машине (`ZombieAI` + `NavMeshAgent`).  
* **Замедление грузовика** при контакте коллайдеров — агрегация через `VehicleCrowdBrake` (`ZombieCrowdResistance`).  
* **Сбивание** — при достаточной планарной скорости грузовика зомби получает заглушку смерти (`ZombieDeathHandler.Die`, без регдолла).  
* **HP грузовика** — компонент `VehicleHealth` на корне с **Rigidbody**: максимум **100 HP**, поражение при **0 HP**; газ отключается через `CarControl`. Пока зомби в контакте (зарегистрирован в `VehicleCrowdBrake`), каждый зомби наносит машине **5 HP** с интервалом **2 с** (параметры в `ZombieStats`: `damageToVehiclePerHit`, `vehicleDamageHitIntervalSeconds`).  
* Остановка агента на дистанции до пивота игрока + **extra clearance** — снижение постоянного «вдавливания» в кузов и толчка от NavMesh.

### **Известные проблемы / риски**

* **Толчок машины зомби** — возможен при **твёрдом** коллайдере и неудачной настройке дистанции остановки / слоёв; требует подгонки `Stop Chasing Player Distance`, `Extra Clearance From Truck Pivot` и матрицы **Truck ↔ Zombie**.  
* **Сбивание «не срабатывает»** — проверять порог `runOverSpeedKmh`, тег **Player** на корне с **Rigidbody**, наличие **не-триггерного** коллайдера у зомби (иначе нет физического сбивания по скорости в текущей схеме), назначение **Car Stats** на `VehicleCrowdBrake`.

### **Отложено (по запросу)**

* **Animator Controller** (параметр **Speed**, Idle/Run).  
* **Surrounded Lose Timer** / окружение при остановке.  
* **Цепляние к кузову** — отдельная механика, в архитектуре зарезервировано (`ZombieAttachmentSystem`, лимит **5**).

---

## **Статус: текущий объём vs будущее**

**Сейчас (прототип / первая итерация):** зомби **бегут к автомобилю игрока** (один игрок в сцене). При **столкновении с авто** толпа **замедляет** машину; при контакте наносится **урон по HP** грузовика по правилам выше. Игрок оценивает силы и **избегает въезда в большие толпы**. Навигация с **обходом препятствий** (см. ниже). Смерть зомби — **без регдолла**, временная **заглушка**.

**Отдельной последующей механикой:** **цепляние зомби к кузову** — до **5** активно висящих на машине, сброс при манёврах и т.д.

## **🔷 Общее описание**

Зомби — противники на уровне. Расставляются заранее (и при необходимости через спавнер). Когда грузовик попадает в **радиус активации**, зомби **преследуют** его.

**Навигация и столкновения между агентами**  
Нужно, чтобы зомби **обходили препятствия**: здания, столбы, **других зомби**. Реализация: **NavMesh** + радиус агента и при необходимости **RVO / separation**, чтобы модели **не пересекались** и не «застывали» внутри друг друга.

**Столкновение с авто**  
* Замедление — вклад в `VehicleCrowdBrake`.  
* **Урон по HP** — пока контакт активен (`ZombieCrowdResistance` + `VehicleHealth`).  
* **Сбивание** — при высокой скорости грузовика — `ZombieDeathHandler.Die` (заглушка).

**Коллайдер зомби и грузовик**  
* **Триггер** — грузовик **проезжает сквозь** зомби (нет силы отталкивания).  
* **Не триггер** — **столкновение** с кузовом; при этом нужно настроить остановку агента (**stop distance / clearance**), иначе возможен **толчок** от постоянного пересечения с NavMesh.

**Поражение**  
* **HP грузовика = 0** — поражение (`VehicleHealth.OnDestroyed`; подписка: UI, смена сцены, `LoseHandler`).  
* Дополнительно (когда будет реализовано): окружение при остановке — только зомби на земле, таймер (см. механику транспорта).

## **🔷 Параметры (ZombieStats и смежное)**

| Параметр | Тип | Описание | Пример |
| ----- | ----- | ----- | ----- |
| Количество HP | float | Здоровье зомби | 20–100 |
| Speed | float | Скорость бега (агент) | 10–50 |
| Radius | float | Радиус активации | 50–100 |
| Stop Chasing / Extra Clearance | float | Остановка до пивота + запас от кузова | см. ассет |
| Замедление авто | float | Вклад в `VehicleCrowdBrake` | 0.1–1 |
| **Damage To Vehicle Per Hit** | float | Урон по HP грузовика за тик | **5** |
| **Vehicle Damage Hit Interval** | float | Секунды между ударами по машине | **2** |
| Run Over Speed Kmh | float | Порог скорости сбивания | настраивается |
| Цепляемость | float | Резерв под будущую механику | 0.1–1 |
| Масса | float | Вес / настройки агента | 10–100 |

**Транспорт (`VehicleHealth`):** **Max Health** = **100** (по умолчанию в компоненте).

## **🔷 Будущая механика: цепляние**

При **низкой скорости** контакт может переводить зомби в режим **прикреплён к кузову** — **не более 5** одновременно. Ветка **не активна** до `ZombieAttachmentSystem` + `ZombieAttachBehaviour`.

---

## **🔷 Идеи**

### 1) Цепляние к кузову (расширенная версия)
- При контакте на малой/средней скорости зомби не умирает, а переходит в состояние «прилип к машине».
- Ограничение по количеству активных прицепившихся зомби (например 5), превышение даёт усиленный дебафф скорости/управления.
- Сброс зомби с кузова через манёвры (резкий поворот, дрифт, удар об препятствие) и/или отдельную кнопку действия.

### 2) Физические трупы и затруднение проезда
- После смерти зомби переходит в **ragdoll** и остаётся в сцене ограниченное время.
- Трупы имеют физический коллайдер и влияют на проезд: машина теряет скорость, повышается риск заноса.
- Нужен лимит активных физических трупов (пул/деспаун), чтобы не просаживать производительность.

### 3) Локальный HP по частям тела (dismemberment)
- HP зомби распределяется по зонам/частям тела (голова, руки, ноги, торс), а не одним общим числом.
- При критическом уроне часть тела отделяется, становится отдельным физическим объектом и остаётся на земле.
- Отделённые части также участвуют в физике мира (мелкие препятствия), но с бюджетом по количеству активных объектов.
- Для дизайна: части тела могут по-разному влиять на поведение зомби до смерти (например, потеря ноги = сильное замедление).
